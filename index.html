軟體操作（深）

架構總覽

此專案採用 Turborepo Monorepo 架構，包含前台、後台與共用套件。

📦 apps/
├── case-management/ # 主前台
└── admin/ # 後台管理
📦 packages/
├── ui/ # 共用元件
└── config/ # ESLint / TS 設定

資料庫與 API 整合

  • 使用 Supabase 作為主要資料庫，RLS 規則需逐表設定
  • Google Apps Script 作為 Webhook 橋接層，接收表單並寫入 Sheets
  • 前台透過 fetch + CORS headers 存取 GAS doGet endpoint

部署指令（Production）

pnpm turbo run build
vercel deploy --prod --scope=team-xxx

安全性注意事項

  • 金鑰絕不 commit 至 Git，使用 Vercel 環境變數管理
  • GAS 公開端點需加入 token 驗證避免濫用
  • Supabase anon key 限制只讀權限，寫入需使用 service role
